# UI マイクロコピー詳細パターンライブラリ

SKILL.md Section 5 の詳細版。TrendBird の全 UI テキストに適用するコピーパターン。

---

## 1. エラーメッセージ

### 3層構造

すべてのエラーメッセージは以下の3層で構成する:

```
[共感] + [解決策] + [エスカレーション（重度のみ）]
```

| 層 | 役割 | 例 |
|----|------|----|
| 共感 | ユーザーを責めない表現 | 「うまくいきませんでした」 |
| 解決策 | 次にすべきこと | 「しばらくしてから再度お試しください」 |
| エスカレーション | 解決しない場合 | 「改善されない場合はお問い合わせください」 |

### gRPC コード別テンプレート

| gRPC コード | ユーザー向けメッセージ | 解決策 |
|------------|---------------------|--------|
| InvalidArgument | 入力内容に誤りがあります | （具体的なフィールド指示） |
| NotFound | データが見つかりません | — |
| AlreadyExists | 既に登録されています | — |
| PermissionDenied | 権限がありません | — |
| Unauthenticated | 認証が必要です | 再度ログインしてください |
| ResourceExhausted | 利用上限に達しました | — |
| FailedPrecondition | 操作を実行できません | （前提条件を説明） |
| Internal | ただいま処理がうまくいきませんでした | しばらく時間をおいて再度お試しください。改善されない場合はお問い合わせください |
| Unavailable | 現在サービスに接続しづらい状態です | しばらく時間をおいて再度お試しください |
| DeadlineExceeded | 処理に時間がかかりすぎました | しばらく時間をおいて再度お試しください |

### 操作別エラーテンプレート

ユーザーが起動した操作のエラーには、操作名を含める:

```
{操作名}がうまくいきませんでした。しばらくしてから再度お試しください
```

| 操作 | エラーメッセージ |
|------|---------------|
| 削除 | 削除がうまくいきませんでした。しばらくしてから再度お試しください |
| 投稿 | 投稿がうまくいきませんでした。しばらくしてから再度お試しください |
| 保存 | 保存がうまくいきませんでした。しばらくしてから再度お試しください |
| 接続テスト | 接続テストがうまくいきませんでした。しばらくしてから再度お試しください |
| キャンセル | キャンセル処理がうまくいきませんでした。しばらくしてから再度お試しください |
| アカウント削除 | アカウント削除がうまくいきませんでした。しばらくしてから再度お試しください |

### バリデーションエラー

入力エラーはリアルタイムでフィールド直下に表示:

| パターン | 例 |
|---------|-----|
| 有効な〜を入力してください | 有効なキーワードを入力してください |
| 〜は最大N個までです | ルールは最大5個までです |
| 同じ〜が既に追加されています | 同じキーワードが既に追加されています |
| 〜を入力してください | 投稿文を入力してください |

## 2. 成功メッセージ

### toast パターン

成功メッセージは toast（2秒自動消去）で表示。

**テンプレート:** `{対象}を{動作}しました`

| 操作 | メッセージ |
|------|----------|
| 保存 | ルールを保存しました / 設定を保存しました |
| 作成 | ルールを作成しました |
| 削除 | ルールを削除しました / 削除しました |
| 投稿 | 投稿しました |
| 更新 | 投稿を更新しました |
| 予約 | 投稿を予約しました |
| コピー | コピーしました |
| 下書き | 下書きに保存しました |
| 接続 | X APIに接続しました |

### 成功メッセージのルール

1. **過去形**（「〜しました」）で完了を伝える
2. **対象を明示** — 「保存しました」より「ルールを保存しました」
3. **感嘆符なし** — 落ち着いたトーン
4. **絵文字なし** — テキストのみ

## 3. 空状態

### 初回利用（データ未作成）

ユーザーがまだデータを作成していない状態。行動を促す CTA を含める。

| 画面 | メッセージ | CTA |
|------|----------|-----|
| 下書き一覧 | 下書きはありません | — |
| 予約投稿一覧 | 予約中の投稿はありません | — |
| 投稿履歴 | 投稿履歴はありません | — |
| ジャンル一覧 | ジャンルが登録されていません | ジャンルを追加 |

### 空状態のルール

1. **事実を述べる** — 「まだ〜はありません」
2. **責めない** — 「〜を作成してください」より「〜を追加すると〜できます」
3. **CTA がある場合は行動を促す** — ボタン or リンク

## 4. 確認ダイアログ

### 破壊的操作

取り消せない操作の確認は以下の構造:

```
[タイトル: 操作名]
[説明: 具体的な影響]
[キャンセルボタン] [実行ボタン（赤系）]
```

| 操作 | タイトル | 説明 | 実行ボタン |
|------|---------|------|-----------|
| アカウント削除 | アカウントの削除 | アカウントを削除すると、すべてのデータが完全に消去されます。この操作は取り消せません。 | 削除する |
| ルール削除 | ルールの削除 | このルールを削除してもよいですか？ | 削除する |
| X連携解除 | X連携の解除 | 連携を解除すると、自動投稿や予約投稿が停止します。 | 解除する |

### 確認ダイアログのルール

1. **具体的な影響を明記** — 「削除しますか？」だけでなく「すべてのデータが消去されます」
2. **取り消し不可を明示** — 「この操作は取り消せません」
3. **キャンセルボタンをデフォルトフォーカス** — 誤操作防止
4. **実行ボタンは赤系（破壊的操作）** — 視覚的な警告

## 5. フォームコピー

### ラベル

- 名詞句で記述: 「テンプレートメッセージ」「トリガーキーワード」
- 「入力してください」等の動詞は含めない

### プレースホルダ

- 入力例 or 動詞句: 「キーワードを入力してEnter」「自動送信するDMのテンプレートを入力」
- 長い説明はヘルプテキストに分離: 「自動送信するDMのテンプレートを入力」

### ヘルプテキスト

- フィールド下に小さい文字で補足
- 「有効にするとキーワードマッチ時にDMを自動送信します」

## 7. チュートリアルコピー

### 基本トーン

チュートリアルは**会話調・励まし**のトーン。

| パターン | 例 |
|---------|-----|
| 〜してみましょう！ | クリックしてトピックを追加してみましょう！ |
| 〜してみよう | ジャンルを選ぼう |
| 〜できます | スパイク・上昇中・安定の件数をリアルタイムで確認できます |
| 〜をクリック | 「テクノロジー」をクリックして、ジャンルを追加しましょう！ |

### Welcome メッセージ

```
TrendBird へようこそ！
X(Twitter)のトレンドを自動モニタリングし、
盛り上がりをリアルタイムでキャッチ。
AI投稿文の生成も可能です。
まずは監視したいトピックを設定しましょう！
```

### 完了メッセージ

```
セットアップ完了！
TrendBirdの準備が整いました！
[始める]
```

### チュートリアルコピーのルール

1. **会話調** — 「〜してみましょう」「〜してみよう」
2. **励まし** — 「！」を適度に使用
3. **短文** — 1ステップの説明は2文以内
4. **具体的な指示** — 「次へ進む」ではなく「サイドバーの『トピック』をクリック」

## 8. 設定画面コピー

### セクションヘッダー

| セクション | ヘッダー | 補足テキスト |
|-----------|---------|------------|
| プロフィール | プロフィール | Xプロフィールから取得 |
| 通知 | 通知 | — |
| X連携 | X連携 | — |
| アカウント | アカウント | — |

### トグル設定

| 設定 | ラベル | 説明文 |
|------|--------|--------|
| 盛り上がり通知 | 盛り上がり通知 | トピックが盛り上がったときにプッシュ通知を受け取ります |
| 上昇中通知 | 上昇中通知 | トピックが上昇傾向にあるときに通知を受け取ります |

### 設定コピーのルール

1. **ラベルは名詞句** — 「盛り上がり通知」
2. **説明文はです/ます調** — 「〜を受け取ります」
3. **トグル = 即時保存** — Save ボタンは不要（Optimistic Update）
