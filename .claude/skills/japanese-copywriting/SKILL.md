## 1. 日本語コピーの6原則

すべてのコピー（UI・LP・X投稿）に適用する基盤ルール。

| # | 原則 | 基準 | 理由 |
|---|------|------|------|
| 1 | **漢字率 30% 以下** | 漢字率が高い文は認知負荷が上がる | ひらがな化で読みやすさ向上 |
| 2 | **1文 40字以内** | 長文は離脱を招く | モバイルで1行半以内 |
| 3 | **丁寧語デフォルト** | です/ます調を基本にする | 信頼感・安心感の担保 |
| 4 | **能動態優先** | 「〜されます」→「〜します」 | 主語明確・行動促進 |
| 5 | **次の行動を示す** | ユーザーが「次に何をすればいいか」を常に明示 | 迷わせない UX |
| 6 | **一貫性** | 同じ概念に同じ言葉を使う（Section 8 参照） | 認知コスト削減 |

### ひらがな化対象リスト

| 漢字表記 | ひらがな表記 | 備考 |
|----------|-------------|------|
| 致します | いたします | |
| 下さい | ください | |
| 出来る | できる | |
| 頂く | いただく | |
| 宜しく | よろしく | |
| 予め | あらかじめ | |
| 殆ど | ほとんど | |
| 尚 | なお | |
| 但し | ただし | |
| 又は | または | |
| 及び | および | |
| 様々 | さまざま | |
| 是非 | ぜひ | |

### トーンスペクトラム

コピーの種類によって丁寧さ・カジュアルさを調整する。

```
LP（親しみ + 期待）
  ↓
チュートリアル（カジュアル + 励まし）
  ↓
アプリ UI（簡潔 + 丁寧）
  ↓
エラーメッセージ（共感 + 解決志向）
  ↓
利用規約・PP（フォーマル）
```

- **LP:** 「〜してみませんか？」「〜を逃しません」— 期待感と親しみ
- **チュートリアル:** 「〜してみましょう！」「〜をクリックしてみよう」— 会話調で励まし
- **アプリ UI:** 「〜しました」「〜できます」— 無駄なく丁寧
- **エラー:** 「うまくいきませんでした」「再度お試しください」— 共感 + 解決策

## 2. パワーワード辞典（概要版）

> 詳細版は `references/power-words.md` を参照

フック・CTA・LP で使う感情を動かす言葉。6カテゴリに分類。

| カテゴリ | 代表ワード | TrendBird 実例 |
|----------|-----------|---------------|
| **緊急性** | 今すぐ、見逃さない、リアルタイム、逃さない | 「バズるタイミングを逃しません」 |
| **安心感** | 無料、かんたん、30秒、クレジットカード不要 | 「クレジットカード不要・30秒で開始」 |
| **低コミット** | まずは、試してみる | 「無料で始める」 |
| **社会的証明** | 話題、注目、みんな、〜人が利用 | 「今Xで話題のトレンドを…」 |
| **共感** | あるある、〜って経験ないですか | 「トレンド追いきれなくなってきた」 |
| **具体数字** | 3ステップ、1日1回、30秒 | 「かんたん3ステップ」「30秒で開始」 |

### 日本市場の鉄則

1. **Trust > FOMO** — 煽りより信頼。「無料・安心・いつでもやめられる」が効く
2. **和（調和）> 煽り** — 強い断定より柔らかい提案。「〜してみませんか」
3. **具体数字 > 曖昧表現** — 「すぐ」より「30秒で」、「たくさん」より「3ステップ」

## 3. CTA・ボタンコピーのルール

### 4原則

| # | 原則 | 説明 | 例 |
|---|------|------|----|
| 1 | **ベネフィット先行** | ユーザーが得るものを動詞で | 「無料で始める」> 「登録する」 |
| 2 | **2-4語** | 短く、一目で理解できる | 「始めてみる」「AI投稿文を生成する」 |
| 3 | **安心コピー直下** | CTA ボタンの直下に不安を解消する一文 | 「クレジットカード不要・30秒で開始」 |
| 4 | **1セクション1CTA** | 選択肢が多いと行動しない | ヒーローに「無料で始める」のみ |

### ソフト CTA

強い「今すぐ購入」ではなく、日本市場向けの柔らかい CTA:

| ソフト CTA | 用途 |
|-----------|------|
| 無料で始める | LP ヒーロー |
| 試してみる | 機能紹介セクション |
| 詳しく見る | FAQ・情報系 |

### TrendBird の既存 CTA パターン

| 場所 | CTA テキスト | 安心コピー |
|------|-------------|-----------|
| LP ヒーロー | 無料で始める | クレジットカード不要・30秒で開始 |
| 設定 X連携 | 接続する / 接続テスト | — |

### ボタンラベルの型

| 操作種別 | パターン | 例 |
|---------|---------|-----|
| 作成 | 〜する / 〜を作成 | 投稿する、ルールを作成 |
| 保存 | 保存する | 保存する |
| 削除 | 削除する | 削除する（確認ダイアログ付き） |
| 切り替え | 有効 / 無効 | トグル UI で即時保存 |
| ナビゲーション | 〜を見る / 〜へ | 詳しく見る、次へ |
| キャンセル | キャンセル | キャンセル |

## 4. LP・セールスコピーのフレームワーク

### 起承転結構造

LP は以下の4パート構造で設計する:

| パート | 役割 | TrendBird LP の実装 |
|--------|------|-------------------|
| **起（Hook）** | スクロールを止める | 「Xのトレンドを検知し、AIが投稿文を自動生成」 |
| **承（Problem）** | 課題の共感 | How it Works で「3ステップで解決」 |
| **転（Solution）** | 具体的解決策 | 機能紹介・FAQ |
| **結（CTA）** | 行動を促す | 「無料で始める」+ 安心コピー |

### ヘッドライン公式4型

| 型 | パターン | TrendBird 例 |
|----|---------|-------------|
| **ベネフィット+ターゲット** | 〜で〜する | 「Xのトレンドを検知し、AIが投稿文を自動生成」 |
| **疑問型** | 〜していませんか？ | 「トレンド、まだ手動で追いかけていませんか？」 |
| **数字型** | N つの〜 | 「かんたん3ステップ」 |
| **Before/After** | 〜が〜に変わる | 「トレンド追いが作業→自動検知に変わる」 |

## 5. UI マイクロコピーパターン集（概要版）

> 詳細版は `references/ui-copy-patterns.md` を参照

### エラーコピー3原則

1. **共感** — ユーザーを責めない。「うまくいきませんでした」（「失敗しました」ではない）
2. **解決策** — 次に何をすればいいか示す。「しばらくしてから再度お試しください」
3. **エスカレーション** — 解決しない場合の連絡先。「改善されない場合はお問い合わせください」

### パターン早見表（12シーン）

| シーン | パターン | TrendBird 実例 |
|--------|---------|---------------|
| **成功** | 〜しました | 「設定を保存しました」 |
| **エラー（ユーザー操作）** | 〜がうまくいきませんでした。しばらくしてから再度お試しください | 「削除がうまくいきませんでした…」 |
| **エラー（サーバー）** | ただいま処理がうまくいきませんでした。しばらく時間をおいて再度お試しください | connect-error.ts のデフォルト |
| **エラー（入力）** | 入力内容に誤りがあります / 有効な〜を入力してください | 「キーワードを入力してください」 |
| **ローディング** | 無言（スピナーのみ）or 〜を読み込んでいます | — |
| **空状態（初回）** | まだ〜がありません + CTA | 「下書きはありません」 |
| **空状態（フィルタ）** | 条件に一致する〜はありません | — |
| **確認（破壊的）** | この操作は取り消せません + 具体的影響 | 「すべてのデータが完全に消去されます」 |
| **確認（重要操作）** | 操作の影響 + 取り消し可否 | — |
| **チュートリアル** | 〜してみましょう！/ 〜してみよう | 「クリックしてトピックを追加してみましょう！」 |
| **フォーム** | ラベル: 名詞句 / プレースホルダ: 動詞句 | 「テンプレートメッセージ」/「キーワードを入力してEnter」 |

### toast メッセージのルール

- **成功:** 「〜しました」（過去形・簡潔）
- **エラー:** 「〜がうまくいきませんでした。しばらくしてから再度お試しください」
- 2秒で自動消去
- 絵文字なし・感嘆符なし

## 6. X投稿向けフック辞典

### 感情トリガー × フックパターン

| 感情トリガー | フックの型 | 例 | 対応パターン |
|-------------|----------|-----|---------------------|
| **驚き** | 「〜って知ってた？」「まさかの〜」 | 「Xの内部アルゴリズム、リプライの重みが150倍って知ってた？」 | 反常識・数字 |
| **共感** | 「〜って経験ないですか」「あるある:」 | 「個人開発、機能は作れるけどマーケティングが永遠の課題…って経験ないですか」 | 共感・本音 |
| **好奇心** | 「なぜ〜なのか」「〜の理由」 | 「なぜ今AIツールが次々出てくるのに、トレンドを追える人は少ないのか」 | 疑問・比較 |
| **FOMO** | 「〜を見逃している」「今〜が起きている」 | 「今Xで起きてるトレンド、手動で追いかけてたら間に合わない」 | 緊急性・時事 |
| **権威性** | 「〜を分析した結果」「データで見ると」 | 「100件のバズ投稿を分析した結果、共通点は3つだった」 | 数字・分析 |

### 日本市場向け注意点

1. **FOMO は控えめに** — 「見逃すな！」より「知っておくと便利です」
2. **権威性は謙虚に** — 「私が教えます」より「調べてみたら面白かった」
3. **共感が最強** — 日本のXでは共感 > 情報価値。「わかる」を引き出すフック
4. **驚きは数字で裏付ける** — 「すごい」より「150倍」。具体数字で説得力

### フック選択ガイド

| 投稿タイプ | 推奨トリガー | 理由 |
|-----------|------------|------|
| TrendBird 告知 | 好奇心 + 共感 | プロダクトへの興味 + 課題共感 |
| 開発日記 | 共感 + 驚き | 開発者あるある + 発見の共有 |
| トレンド分析 | 権威性 + 好奇心 | データに基づく独自視点 |
| Tips・ノウハウ | 好奇心 + 驚き | 「知らなかった」感 |
| 本音・失敗談 | 共感 | 圧倒的に共感が刺さる |

## 7. コピー品質チェックリスト

### X投稿（5項目）

| # | チェック項目 | 基準 | 不合格時の対応 |
|---|------------|------|-------------|
| 1 | 漢字率 | 30% 以下 | ひらがな化（Section 1） |
| 2 | 1行の長さ | スマホで折り返さない（全角20字目安） | 改行位置を調整 |
| 3 | パワーワード | 最低1つ含む（Section 2） | フック or CTA にパワーワードを追加 |
| 4 | CTA or 余韻 | 末尾にある | 問いかけ・URL・行動喚起を追加 |
| 5 | 音読テスト | 声に出して自然 | AI臭い表現を口語に変換 |

### UI（4項目）

| # | チェック項目 | 基準 | 不合格時の対応 |
|---|------------|------|-------------|
| 1 | エラーは共感+解決策 | 「うまくいきませんでした」+ 次のアクション | Section 5 のパターンに準拠 |
| 2 | ボタン語数 | 2-4語 | 短縮 or ベネフィット表現に変更 |
| 3 | 用語統一 | Section 8 の正式表記 | 統一表に合わせる |
| 4 | 丁寧語の一貫性 | です/ます調で統一 | トーンスペクトラム（Section 1）に準拠 |

### LP（4項目）

| # | チェック項目 | 基準 | 不合格時の対応 |
|---|------------|------|-------------|
| 1 | ヒーローの明確性 | 3秒で「何ができるか」が伝わる | ベネフィット+ターゲット型に書き直し |
| 2 | CTA の単一性 | 1セクション1CTA | 副次 CTA を削除 |
| 3 | 安心コピーの存在 | 主要 CTA の直下にある | 「無料・カード不要・すぐ開始」等を追加 |
| 4 | FAQ の網羅性 | ユーザーの不安 TOP4 をカバー | 安全性・データ管理・セットアップ・機能範囲 |
